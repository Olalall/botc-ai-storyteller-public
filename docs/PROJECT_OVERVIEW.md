# Project Overview / 项目说明书

## 项目定位

BOTC AI Storyteller Assistant 是一个非官方、免费、本地优先的《血染钟楼》说书人辅助工具。它把说书人端、玩家手机端、房间状态、配板发身份、夜晚技能结果结算候选、投票处决和复盘集中到一个局域网浏览器系统里。

它不是官方 App，也不是商业化平台。项目边界是：帮助说书人减少重复操作和信息整理，不替代说书人的最终判断。

## 设计原则

- **本地优先**：一台电脑启动服务，玩家手机通过同一局域网访问。
- **说书人确认**：所有会改变权威游戏状态的动作都需要说书人明确确认。
- **自动结算有边界**：规则逻辑可以整理玩家行动并生成技能结果结算候选，但说书人确认后才写入权威状态。
- **AI 只起草**：AI/规则逻辑只生成候选、提示和草稿，不直接裁决复杂规则。
- **公开透明**：README 第一屏说明非官方身份；License 只覆盖本项目源码和文档。
- **完整体验**：保留线下开局需要的核心流程和视觉资产，让项目能直接运行演示。

## 适合谁用

1. 线下面团/桌游店，需要更稳定地开房、发身份、跑夜晚流程。
2. 局域网测试组，希望玩家用手机接收身份和私密信息。
3. 想研究 BOTC 辅助工具实现方式的开发者。
4. 想把 AI 用在“候选建议/流程减负”而不是“自动裁决”的产品实践者。

## 当前边界

这个项目目前更像一个可运行的 Beta 原型，而不是成熟云服务：

- 建议本地或局域网使用。
- 不承诺生产级多房间并发、账号系统或长期存档。
- 官方/社区角色和图标相关知识产权不属于本项目。
- 复杂角色逻辑仍需要说书人按现场信息确认。
- AI provider 如果未配置，AI 相关能力应保持草稿/辅助边界。

## 与常见 BOTC 工具的区别

- 更偏“线下说书人操作台”，不是单纯脚本编辑器。
- 保留玩家手机端，不只服务说书人单屏。
- 把自动技能结算放在“候选结果 + 说书人确认”层，不包装成全自动规则裁判。
- 把 AI 放在候选/草稿层，不把 AI 包装成规则裁判。
- 开源仓库明确说明非官方身份、免费分发和知识产权边界。

## 后续方向

1. 增加更多真实桌局截图和短视频演示。
2. 优化自定义 JSON 导入和导入后的人工复核流程。
3. 强化自动技能结果结算说明，让说书人清楚哪些结果是系统候选、哪些必须人工裁定。
4. 改善复杂角色候选解释，让说书人更容易判断。
5. 继续减噪 UI，把每个阶段的主任务放到更明显的位置。
6. 进一步拆分源码、运行资产和第三方 IP 声明。
