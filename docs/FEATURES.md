# Feature Guide / 核心功能说明

## 1. 说书人工作台

说书人端是项目的主入口，目标是把线下主持一局 BOTC 时最常用的动作集中起来：

- 创建房间、查看房间码和玩家加入状态。
- 管理座位、玩家名称、存活/死亡等公开状态。
- 选择剧本、生成配板候选、确认发身份。
- 查看夜晚顺序、当前角色提示和待确认事项。
- 整理玩家提交的夜晚行动，生成技能结果结算候选。
- 管理提名、投票、处决和昼夜切换。
- 进入复盘和查看关键局面记录。

核心原则：说书人端是权威操作端，工具只辅助，不绕过说书人确认。

## 2. 玩家手机端

玩家端用于手机浏览器：

- 输入房间码加入。
- 认领座位。
- 接收自己的身份。
- 查看可公开或可私密展示给自己的信息。
- 在需要时提交夜晚行动或选择目标。

玩家端只能看到自己的 PlayerView，不应该看到全局魔典、其他玩家真实身份或超出规则允许的信息。

## 3. 配板与发身份

配板流程强调“候选 + 确认”：

- 工具可以根据人数和剧本生成候选分配。
- 说书人需要确认后才进入发身份。
- 身份发出后，本局配板锁定，不能静默重置或切换剧本。

这样避免误点导致已开局信息被破坏。

## 4. 夜晚流程

夜晚流程围绕当前角色和待处理动作展开：

1. 显示当前夜晚阶段。
2. 按夜序提示当前角色。
3. 玩家端可提交必要行动。
4. 系统整理候选目标或候选结果。
5. 根据已知信息生成技能结果结算候选。
6. 说书人确认、修改或改为手动裁定。
7. 确认后再发送玩家可见结果或写入魔典状态。
8. 进入下一位或结束夜晚。
9. 记录关键公开/私密变化。

复杂角色、毒醉、错误信息、恶魔伪装等内容不应由工具自动裁决，仍由说书人按局面判断。

## 5. 自动技能结果结算候选

项目里“自动结算”指的是辅助说书人把可整理的信息先算成候选结果，而不是绕过说书人自动改局面。

它重点解决这些重复操作：

- 汇总玩家夜晚行动和目标选择。
- 按夜序把待处理技能排列出来。
- 对可计算的结果生成候选，例如信息提示、死亡/存活候选、处决门槛或无处决候选。
- 标记“待说书人确认”的结果，避免直接写入权威状态。
- 支持说书人修改、覆盖或使用手动处理工具裁定边缘情况。
- 确认后再把结果发送给对应玩家，或更新公开/私密状态。

推荐对外介绍时使用这个说法：

```text
Automatic skill-result settlement candidates, storyteller-confirmed final state.
自动整理技能结果候选，说书人确认后才生效。
```

## 6. 白天、提名和投票

白天阶段用于管理提名和投票：

- 发起提名。
- 记录当前被提名人。
- 跟踪票数、投票资格和是否达到处决条件。
- 由说书人确认处决或不处决。
- 切换到下一夜或进入游戏结束复盘。

## 7. 剧本和导入

当前公开包保留基础运行所需数据，并支持官方基础剧本、Catfishing 等已整理剧本数据。对外使用时应保持两个边界：

- 本项目 License 不覆盖角色名、技能文字、图标、剧本内容等 BOTC IP。
- 用户自行导入 JSON 或社区剧本时，应由用户确认来源和使用权限。

## 8. AI 辅助边界

AI 不是裁判，也不是自动说书人。

```text
AI / Rule automation = 候选 / 草稿 / 技能结果结算建议 / 提醒
Storyteller = 确认 / 裁决 / 写入权威状态
```

如果没有配置 AI provider，AI 相关能力应降级为普通规则提示或保持不可用，不影响本地核心流程。

## 9. 本地 / 局域网运行

默认运行方式：

```bash
npm install
npm start
```

同一局域网内，玩家手机访问主机 IP 对应的玩家页面即可。例如主机 IP 是 `192.168.1.23` 时：

```text
http://192.168.1.23:3000/player-v2.html
```

## 10. 推荐演示路径

1. 启动本地服务。
2. 说书人打开 `storyteller-v2.html` 创建房间。
3. 玩家手机或第二个浏览器打开 `player-v2.html` 加入。
4. 选择人数、填充测试玩家或真实玩家。
5. 生成配板候选，确认后发送身份。
6. 进入首夜，演示夜晚技能结果候选生成、说书人修改/确认和结果生效边界。
7. 进入白天，演示提名、投票和处决确认。
8. 打开角色库、AI 辅助和复盘入口。

## 11. 当前不承诺

- 不保证复杂角色自动裁决完全正确；自动结算只应理解为“结果候选生成 + 说书人确认”。
- 不提供官方授权内容使用许可。
- 不提供生产级账号、支付、多租户和云存档能力。
- 不替代线下说书人判断。
